Operační systémy

Programové vybavení

Programové vybavení (software) je nezbytné pro využití technického vybavení (hardware). V počátcích počítačové éry byly programy vytvářeny vždy pro konkrétní hardware. S rozvojem sériové výroby počítačů vznikla potřeba vývoje systémových programů, které by obstaraly základní operace s hardwarem a poskytly prostředí pro běh aplikací.

Operační systém

Základním programovým vybavením počítačů jsou operační systémy (OS Operating System). Jejich nejdůležitější funkcí je řízení hardware. To zajišťuje jádro systému, které obsahuje subsystémy pro správu procesoru, paměti, diskových médií a dalších zařízení. Jádro vyžaduje instalaci ovladačů (drivers), které zpřístupňují hardware běžícím procesům.

API a uživatelské rozhraní

Nezbytnou součástí OS je aplikační rozhraní (API Application Programming Interface), které poskytuje programátorům prostředky pro vývoj aplikací. Uživatelské rozhraní slouží k ovládání OS a spouštění programů. Starší systémy používaly příkazový řádek (CLI - Command Line Interface), moderní systémy využívají grafické rozhraní (GUI - Graphic User Interface).

Historie operačních systémů

Na počátku vývoje moderních OS stál víceúlohový a víceuživatelský systém UNIX, používaný od 70. let v sálových a minipočítačích. Jeho principy ovlivnily mnoho dalších OS. V 80. letech soupeřily systémy MS DOS (Microsoft) a Mac OS (Apple), který jako první masově využil grafické rozhraní. To se později stalo základem MS Windows.

Velkou popularitu získal také volně šiřitelný systém Linux, pojmenovaný po svém tvůrci Linusi Torvaldsovi. Zatímco Windows dominují na osobních počítačích, různé linuxové distribuce jsou často instalovány na síťových serverech.

Typy operačních systémů

  • Serverové systémy: MS Windows Server, Linux, Unix
  • Desktopové systémy: MS Windows, Linux Ubuntu, macOS
  • Mobilní systémy: Android, iOS, dříve Symbian a Windows Phone
  • RTOS - systémy reálného času pro zařízení, kde je nutná okamžitá reakce (např. robotické kamery, měřicí přístroje, průmyslové řízení)

Zatímco u běžných desktopových OS lze některé operace odložit a provádět paralelně, RTOS musí pracovat bez prodlevy a s vysokou spolehlivostí.
